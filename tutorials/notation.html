Markdown / HTML / GitHub Pages

記法メモ

READMEやdocsではMarkdownを使い、GitHub PagesではHTML/CSS/JavaScriptを使います。 このページでは、ポートフォリオ全体で表記をそろえるための基本記法を整理します。

今回のGitHub Pagesについて 現在のKnowledgeサイトはMarkdownではなく、静的なHTML/CSS/JavaScriptで作成しています。GitHub Pages自体はMarkdown/Jekyllでも作れますが、今回は見た目とレイアウトを細かく制御しやすいHTML構成にしています。

Markdownの基本

用途 記法 使用場所
見出し #、##、### README、docs
リンク [表示名](URL) README、docs
画像 ![代替テキスト](path/to/image.png) README、docs
コード バッククォート、またはフェンスコードブロック README、docs
表 | 列1 | 列2 | README、docs

見出し

READMEでは、リポジトリ名をH1、主要セクションをH2、手順内の小項目をH3にします。

markdown
# aws-example-repository

## 概要

## アーキテクチャ

## デプロイ方法

### スタック作成

### スタック削除

リンク

同じリポジトリ内のファイルへは相対リンクを使います。GitHub上でクリックして遷移できるよう、コードブロックではなくMarkdownリンクにします。

markdown
[CloudFormationテンプレート](templates/example.yml)

[パラメータファイル](parameters/example.json)

[デプロイ手順](docs/deployment.md)

コードブロック

コマンド例は言語名を付けて記載します。PowerShellの場合は powershell、YAMLの場合は yaml を指定します。

markdown
```powershell
aws sts get-caller-identity
```

```yaml
AWSTemplateFormatVersion: '2010-09-09'
Description: Example template
```

表

パラメータ、作成リソース、手動作業は表にすると比較しやすくなります。

markdown
| パラメータ | 説明 | デフォルト |
| --- | --- | --- |
| ProjectName | リソース名の接頭辞 | example |
| EnvironmentName | 環境名 | dev |

プレースホルダー

利用者が自分の値に置き換える箇所は、山括弧を使って明示します。

text
<YOUR_STACK_NAME>
<YOUR_KEY_PAIR_NAME>
<YOUR_NOTIFICATION_EMAIL>

Pages側のHTML記法

このサイトでは、各ページをHTMLで作成します。見出しには目次リンク用の id を付けます。

html
<h2 id="deploy">デプロイ方法</h2>

<p>
  ここに説明文を記載します。
</p>

表記ルール

  • AWSサービス名は正式名称を優先します。
  • ファイルパスはMarkdownリンクにします。
  • コマンド例はコードブロックにします。
  • 利用者が置換する値は <YOUR_...> 形式にします。
  • READMEには概要、docsには詳細手順、Knowledgeには横断的な手順を記載します。
