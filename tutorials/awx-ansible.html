Automation / Configuration Management

AWX / Ansible 手順

AWXでAnsible Playbookを実行するための基本要素と、ジョブ実行時の確認観点を整理します。

主要コンポーネント

項目役割
Inventory管理対象ホストを定義します。
CredentialSSH鍵、ユーザー、Vault Passwordなどの認証情報を管理します。
ProjectGitリポジトリ上のPlaybookを同期します。
Job TemplateInventory、Credential、Playbook、Variablesを組み合わせて実行単位を定義します。

Playbook実行

yaml
- name: Check Linux uptime
  hosts: linux
  gather_facts: false
  tasks:
    - name: Run uptime
      ansible.builtin.command: uptime
      changed_when: false

実行ログ確認

  • `ok`、`changed`、`failed`、`skipped` の件数を確認します。
  • 変更を伴わない確認タスクは `changed_when: false` を設定します。
  • 失敗時は対象ホスト、モジュール、戻り値、標準エラーを確認します。
