[構築] Red Hat Enterprise Linux 9

AWS EC2上のRed Hat Enterprise Linux 9を手動構築する際の初期設定、SSH、cloud-init、swap、ミドルウェア、Apache、Postfix、logrotateを整理します。root権限で実行する前提のため、コマンド例では sudo を省略しています。

ℹ️ 前提 実環境名、IPアドレス、顧客固有値は扱わず、再利用できる手順として一般化しています。<HOSTNAME>、<FQDN>、<PRIVATE_IP> などは環境に合わせて置き換えます。

1. 起動時の確認

AWSコンソールでEC2を起動する場合、キーペアは ED25519 で作成します。RSA鍵は組織ポリシーや鍵長の条件で拒否される場合があります。

確認項目内容
キーペアED25519 を利用します。
接続方式SSH、Session Manager、踏み台経由など、作業経路を事前に確認します。
バックアップ設定変更前にAMI、EBS Snapshot、設定ファイル退避の要否を確認します。

2. SELinuxの設定変更

Red Hat Enterprise Linux 9では SELINUX=disabled にすると起動や動作に影響する場合があるため、ログ出力のみの permissive に変更します。

設定ファイルを退避します。

root shell
cp -pi /etc/selinux/config /etc/selinux/config.org

設定ファイルを編集します。

root shell
vi /etc/selinux/config

SELINUX を permissive に変更します。

conf
SELINUX=permissive

差分を確認します。

root shell
diff -u /etc/selinux/config.org /etc/selinux/config

再起動前に一時反映します。

root shell
setenforce permissive

動作状態を確認します。

root shell
getenforce

SELinuxのモードは SELinuxのモード解説 も確認します。

3. subscription-managerの無効化

AWS上のRHELでは、環境によって subscription-manager の警告がパッケージ操作時に表示されるため、yumプラグインを無効化します。

設定ファイルを退避します。

root shell
cp -pi /etc/yum/pluginconf.d/subscription-manager.conf /etc/yum/pluginconf.d/subscription-manager.conf.org

設定ファイルを編集します。

root shell
vi /etc/yum/pluginconf.d/subscription-manager.conf

enabled を 1 から 0 に変更します。

conf
[main]
enabled=0
ℹ️ 補足 有効なままだと This system is not registered with an entitlement server. のようなメッセージが表示され、パッケージ更新時の確認が煩雑になる場合があります。

4. ログインプロファイル設定

操作履歴に時刻を残し、調査時に作業順序を追いやすくするため、/etc/profile と新規ユーザー用の雛形を変更します。

/etc/profile を退避します。

root shell
cp -pi /etc/profile /etc/profile.org

/etc/profile を編集します。

root shell
vi /etc/profile

既存内容をベースに、履歴件数、履歴時刻、umaskを設定します。

/etc/profile の設定例を展開する
conf
# /etc/profile
pathmunge () {
    case ":${PATH}:" in
        *:"$1":*)
            ;;
        *)
            if [ "$2" = "after" ] ; then
                PATH=$PATH:$1
            else
                PATH=$1:$PATH
            fi
    esac
}

if [ -x /usr/bin/id ]; then
    if [ -z "$EUID" ]; then
        EUID=`id -u`
        UID=`id -ru`
    fi
    USER="`id -un`"
    LOGNAME=$USER
    MAIL="/var/spool/mail/$USER"
fi

if [ "$EUID" = "0" ]; then
    pathmunge /sbin
    pathmunge /usr/sbin
    pathmunge /usr/local/sbin
else
    pathmunge /usr/local/sbin after
    pathmunge /usr/sbin after
    pathmunge /sbin after
fi

HOSTNAME=`/bin/hostname 2>/dev/null`
HISTSIZE=50000
HISTFILESIZE=50000
if [ "$HISTCONTROL" = "ignorespace" ] ; then
    export HISTCONTROL=ignoreboth
else
    export HISTCONTROL=ignoredups
fi
export PATH USER LOGNAME MAIL HOSTNAME HISTSIZE HISTFILESIZE HISTCONTROL

if [ $UID -gt 199 ] && [ "`id -gn`" = "`id -un`" ]; then
    umask 002
else
    umask 022
fi

for i in /etc/profile.d/*.sh ; do
    if [ -r "$i" ]; then
        if [ "${-#*i}" != "$-" ]; then
            . "$i"
        else
            . "$i" >/dev/null 2>&1
        fi
    fi
done

unset i
unset -f pathmunge
umask 002
HISTTIMEFORMAT='%Y-%m-%d %T '

新規ユーザー作成時の雛形を退避します。

root shell
cp -pi /etc/skel/.bash_profile /etc/skel/.bash_profile.org

新規ユーザー用の .bash_profile を編集します。

root shell
vi /etc/skel/.bash_profile

以下の内容に変更します。

conf
# .bash_profile
if [ -f ~/.bashrc ]; then
        . ~/.bashrc
fi

PATH=$PATH:$HOME/.local/bin:$HOME/bin
export PATH
HISTTIMEFORMAT='%Y-%m-%d %T '
PS1="[\u@\h \W \t]\\$ "
⚠️ 注意 /etc/skel/.bash_profile は新規ユーザー作成時の雛形です。既に作成済みのユーザーには反映されないため、必要に応じて /home 配下の各ユーザーのファイルを個別に修正します。

5. ホスト名とhosts設定

必要に応じてホスト名を変更します。

root shell
hostnamectl set-hostname <HOSTNAME>

ホスト名を確認します。

root shell
hostnamectl

/etc/hosts を編集します。

root shell
vi /etc/hosts

プライベートIP、FQDN、ホスト名を追記します。

conf
<PRIVATE_IP> <FQDN> <HOSTNAME>

6. タイムゾーン設定

サーバーのタイムゾーンを日本時間に変更します。

root shell
timedatectl set-timezone Asia/Tokyo

反映状態を確認します。

root shell
timedatectl status

7. sshd_config修正

SSHのrootログイン、タイムアウト、DNS逆引き、SFTPログ出力を設定します。

設定ファイルを退避します。

root shell
cp -pi /etc/ssh/sshd_config /etc/ssh/sshd_config.org

設定ファイルを編集します。

root shell
vi /etc/ssh/sshd_config

以下の設定を反映します。

conf
#PermitRootLogin prohibit-password
PermitRootLogin forced-commands-only

ClientAliveInterval 60

UseDNS no

#Subsystem      sftp    /usr/libexec/openssh/sftp-server
Subsystem sftp /usr/libexec/openssh/sftp-server -f LOCAL3 -l VERBOSE

sshdを再起動して反映します。

root shell
systemctl restart sshd.service

PermitRootLogin prohibit-password と without-password の扱いは PermitRootLoginの解説 も確認します。

9. 追加ディスクのマウント

EBSボリュームを追加する場合は、ファイルシステム作成、マウントポイント作成、/etc/fstab 追記、再起動後確認まで実施します。詳細は Amazon EBSボリュームの使用 を確認します。

10. swap領域作成

現在のswap領域を確認します。

root shell
free | grep Swap | awk '{print $2}'

ルートファイルシステムの空き容量を確認します。

root shell
df -h | grep /$ | awk '{print $4}'

swapファイルを作成し、権限を変更します。

root shell
dd if=/dev/zero of=/.swapfile bs=512M count=4
chmod 600 /.swapfile

swapを有効化します。

root shell
mkswap /.swapfile
swapon /.swapfile

swap領域を再確認します。

root shell
free | grep Swap | awk '{print $2}'

/etc/fstab を編集します。

root shell
vi /etc/fstab

永続化用の行を追記します。

fstab
/.swapfile    swap    swap    defaults    0    0

11. ミドルウェアのインストール方針

インストールしたいミドルウェアのバージョンがOS標準リポジトリから導入できない場合でも、まずApplication Streamsで選択できないか確認します。いきなり外部リポジトリを追加しない方針にします。

root shell
dnf module list <MIDDLEWARE_NAME>

PostgreSQLの例です。

root shell
dnf module list postgresql
dnf module enable postgresql:15
dnf module list postgresql
dnf module install postgresql/client

パッケージライフサイクルは Application Streams を確認します。Red Hat Enterprise Linux 9で削除・変更されたモジュールは Red Hat Enterprise Linux 9導入時の考慮事項 も確認します。

12. Postfix設定

Postfixをインストールします。SASL認証を利用する場合は cyrus-sasl-plain も導入します。

root shell
yum install postfix
yum install cyrus-sasl-plain

設定ファイルを退避します。

root shell
cp -pi /etc/postfix/main.cf /etc/postfix/main.cf.org
cp -pi /etc/postfix/master.cf /etc/postfix/master.cf.org

main.cf を編集します。

root shell
vi /etc/postfix/main.cf

ホスト名とドメインを設定します。

conf
myhostname = <FQDN>
mydomain = <DOMAIN>

master.cf を編集します。

root shell
vi /etc/postfix/master.cf
/etc/postfix/master.cf の設定例を展開する

13. Apache設定

Apache関連パッケージを導入します。

root shell
yum install httpd.x86_64 httpd-devel.x86_64 httpd-filesystem.noarch httpd-tools.x86_64
yum install mod_ssl

設定ファイルを退避します。

root shell
cp -pi /etc/httpd/conf/httpd.conf /etc/httpd/conf/httpd.conf.org

httpd.conf を編集します。

root shell
vi /etc/httpd/conf/httpd.conf
event MPMのhttpd.conf設定例を展開する
conf
ServerRoot /etc/httpd
PidFile /var/run/httpd/httpd.pid
Timeout 300
KeepAlive On
MaxKeepAliveRequests 100
KeepAliveTimeout 120
User apache
Group apache
AccessFileName .htaccess
<Files ".ht*">
  Require all denied
</Files>
HostnameLookups Off
Include conf.modules.d/*.conf
<IfModule mpm_event_module>
    StartServers             3
    MinSpareThreads         75
    MaxSpareThreads        250
    ThreadsPerChild         25
    MaxRequestWorkers      400
    MaxConnectionsPerChild   0
</IfModule>
<IfModule dir_module>
  DirectoryIndex index.html index.html.var
</IfModule>
<IfModule log_config_module>
    LogFormat "%h %l %u %t \"%r\" %>s %b" common
    LogFormat "%{X-Forwarded-For}i %h %l %u %t \"%r\" %>s %b \"%{Referer}i\" \"%{User-Agent}i\" %D %P" combined
    <IfModule logio_module>
      LogFormat "%h %l %u %t \"%r\" %>s %b \"%{Referer}i\" \"%{User-Agent}i\" %I %O" combinedio
    </IfModule>
</IfModule>
SetEnvIf X-Forwarded-Proto "https" https
CustomLog "/var/log/httpd/access_log" combined env=!https
CustomLog "/var/log/httpd/ssl_access_log" combined env=https
LogLevel warn
ErrorLog "/var/log/httpd/error_log"
<IfModule mime_module>
  TypesConfig /etc/mime.types
  AddType application/x-compress .Z
  AddType application/x-gzip .gz .tgz
  AddType text/html .shtml
  AddOutputFilter INCLUDES .shtml
</IfModule>
<IfModule mime_magic_module>
  MIMEMagicFile conf/magic
</IfModule>
<IfModule mod_negotiation.c>
  LanguagePriority ja en ca cs da de el eo es et fr he hr it ko ltz nl nn no pl pt pt-BR ru sv tr zh-CN zh-TW
  ForceLanguagePriority Prefer Fallback
</IfModule>
AddDefaultCharset off
ServerTokens Prod
ServerSignature Off
EnableSendfile on
SSLProtocol All -SSLv2 -SSLv3
IncludeOptional /etc/httpd/conf.d/*.conf
ℹ️ 補足 prefork MPMはPHPとのモジュール連携などで注意点が多いため、原則としてevent MPMを前提にします。要件上preforkを使う場合は、PHP周辺設定も含めて別途検証します。

/etc/httpd/conf.d/ 配下の不要ファイルを削除します。

root shell
cd /etc/httpd/conf.d/
rm -i autoindex.conf README ssl.conf welcome.conf

文字コード設定ファイルを作成します。

root shell
vi /etc/httpd/conf.d/charaset.conf
conf
AddDefaultCharset Off

セキュリティ設定ファイルを作成します。

root shell
vi /etc/httpd/conf.d/security.conf
conf
ServerTokens Prod
ServerSignature Off
TraceEnable Off
FileETag none

VirtualHost読込設定を作成します。

root shell
vi /etc/httpd/conf.d/vhosts.conf
conf
Listen *:80
Include conf/vhosts/*.conf

設定ファイルの権限を変更します。

root shell
chmod 644 vhosts.conf charaset.conf security.conf

VirtualHost用ディレクトリを作成します。

root shell
mkdir /etc/httpd/conf/vhosts
chmod 755 /etc/httpd/conf/vhosts

VirtualHost設定を作成します。

root shell
cd /etc/httpd/conf/vhosts
vi <FQDN>.conf
conf
<VirtualHost *:80>
  ServerAdmin webadmin@<FQDN>
  DocumentRoot /var/www/<FQDN>/httpd
  ServerName <FQDN>
  SetEnvIf X-Forwarded-Proto "https" https
  ErrorLog /var/log/httpd/<FQDN>.error_log
  CustomLog /var/log/httpd/<FQDN>.access_log combined env=!https
  CustomLog /var/log/httpd/<FQDN>.ssl_access_log combined env=https
  <Directory /var/www/<FQDN>/httpd>
    Options -Indexes
    AllowOverride all
    Require all granted
  </Directory>
</VirtualHost>

必要に応じてviでFQDN部分を一括置換します。

vi
%s/FQDN/<REPLACE_STRING>/g

VirtualHost設定の権限を変更します。

root shell
chmod 644 <FQDN>.conf

ドキュメントルートを作成します。

root shell
mkdir -p /var/www/<FQDN>/httpd
chown -R apache:www /var/www/<FQDN>

構文とVirtualHost一覧を確認してから、httpdを再起動します。

root shell
httpd -t
httpd -tS
systemctl stop httpd.service
systemctl start httpd.service

自動起動を確認し、有効化します。

root shell
systemctl is-enabled httpd.service
systemctl enable httpd.service
systemctl is-enabled httpd.service

14. 基本コマンドのインストール

運用時に利用頻度が高いコマンドを導入します。タブ補完が反映されない場合は再ログイン、必要に応じて再起動後に確認します。

root shell
yum install telnet lftp mlocate libyaml wget mtr tcpdump tmux bash-completion net-tools

15. logrotate設定

既存のlogrotate設定を退避します。

root shell
cp -pi /etc/logrotate.conf /etc/logrotate.conf.org

logrotate設定を編集します。

root shell
vi /etc/logrotate.conf

/etc/logrotate.conf を以下の内容で上書きします。

conf
# see "man logrotate" for details
weekly
rotate 54
create
dateext
compress
include /etc/logrotate.d
# system-specific logs may be also be configured here.
⚠️ 注意 /etc/logrotate.d/ 配下にバックアップファイルを置くと、バックアップファイル内の記述もlogrotateの対象として読み込まれる可能性があります。退避ファイルは別ディレクトリ、またはlogrotateが読み込まない名前・場所で管理します。

16. 作業後確認

logrotateのdry runで、構文と対象ファイルを確認します。

root shell
logrotate -d /etc/logrotate.conf

サービス再起動を伴う設定は、ステータス確認、ログ確認、再ログイン、必要に応じてOS再起動後の確認まで実施します。
