RHEL9 運用手順

AWS EC2上のRHEL9を手動構築・保守する際の確認観点を整理します。実環境名、IPアドレス、顧客固有値は扱わず、再利用できる作業パターンだけをまとめます。

方針 手動手順はAnsible化する前の確認用として扱います。root権限で実行する前提のため、コマンド例では sudo を省略しています。コピーしやすさを優先し、プロンプト記号の # はコードブロックに含めません。

作業前確認

観点確認内容
対象確認アカウント、リージョン、インスタンスID、OSバージョン、ホスト名を確認します。
退避AMI、EBS Snapshot、設定ファイルのバックアップ要否を確認します。
接続経路Session Manager、踏み台、Security Group一時許可のどれを使うか確認します。
影響サービス再起動、パッケージ更新、カーネル更新の有無を確認します。

基本パッケージと確認コマンド

OSバージョンを確認します。

root shell
cat /etc/redhat-release

ホスト名とOS情報を確認します。

root shell
hostnamectl

ネットワークインターフェイスとIPアドレスを確認します。

root shell
ip addr

ディスク容量とメモリ使用状況を確認します。

root shell
df -h
free -m

更新可能なパッケージを確認します。

root shell
dnf check-update

運用確認で利用する基本パッケージをインストールします。

root shell
dnf install -y telnet lftp mlocate wget mtr tcpdump tmux bash-completion net-tools

logrotate設定

ログ保持期間や圧縮有無は環境ごとに差が出ます。変更前に既存設定を退避し、変更後に構文とローテーション対象を確認します。

既存のlogrotate設定を日付付きで退避します。

root shell
cp -pi /etc/logrotate.conf /etc/logrotate.conf.$(date +%Y%m%d)

logrotate設定を編集します。

root shell
vi /etc/logrotate.conf

dry runでlogrotateの実行内容を確認します。

root shell
logrotate -d /etc/logrotate.conf

SSM Agent確認

Session ManagerやPatch Managerを使う場合は、SSM Agent、IAM Role、VPC Endpointまたはインターネット経路を確認します。

SSM Agentの状態を確認します。

root shell
systemctl status amazon-ssm-agent

SSM Agentを自動起動に設定し、再起動します。

root shell
systemctl enable amazon-ssm-agent
systemctl restart amazon-ssm-agent

CloudWatch Agent更新

Agent更新では、更新前バージョン、設定ファイル、更新後のサービス状態、CloudWatch Logsへの到達を確認します。

CloudWatch Agentのインストール済みバージョンを確認します。

root shell
rpm -qa | grep amazon-cloudwatch-agent

CloudWatch Agentを更新します。

root shell
dnf update -y amazon-cloudwatch-agent

更新後のサービス状態を確認します。

root shell
systemctl status amazon-cloudwatch-agent

Apache更新・設定変更

Apacheの脆弱性対応や設定変更では、更新前後のRPM、構文確認、プロセス確認、ステータス確認を残します。

Apacheのインストール済みパッケージを確認します。

root shell
rpm -qa | grep httpd

変更前にApache設定の構文を確認します。

root shell
apachectl configtest

変更前のApacheサービス状態を確認します。

root shell
systemctl status httpd

Apacheを更新します。

root shell
dnf update -y httpd

更新後に設定構文を確認します。

root shell
apachectl configtest

Apacheを再起動し、サービス状態を確認します。

root shell
systemctl restart httpd
systemctl status httpd

作業前後エビデンス

作業前後で同じ観点を比較します。コマンド結果、AWSコンソールの状態、サービス応答、ログ出力を同じ順序で残すとレビューしやすくなります。
