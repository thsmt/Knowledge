Amazon WorkSpaces

Amazon WorkSpaces Personal の構築、ディレクトリ登録、Web Access、カスタムイメージ、Cost Optimizer、トラブル対応の確認観点を整理します

📍 掲載方針

実環境名、ディレクトリID、ユーザー名、グローバルIP、顧客固有のドメイン名は記載せず、再利用できる構築・運用手順として一般化しています

関連リポジトリ: workspaces-ad-foundation-poc

対象範囲

区分 内容
基盤 VPC、Public / Private Subnet、NAT Gateway、VPC Peering、Route Table、Security Group
ディレクトリ AWS Managed Microsoft AD、AD Connector、WorkSpacesディレクトリ登録
WorkSpaces ユーザー選択、バンドル選択、WorkSpace起動、クライアント接続、Web Access
運用 カスタムイメージ、カスタムバンドル、Cost Optimizer、エラー時の切り分け

構築フロー

  1. WorkSpaces用VPCと、AD Connectorまたは管理用接続経路のVPCを設計します
  2. AWS Managed Microsoft ADを作成し、ディレクトリDNS名、NetBIOS名、管理ユーザーを確認します
  3. 必要に応じてAD Connectorを作成し、既存ADまたはManaged Microsoft ADへの接続を構成します
  4. Amazon WorkSpacesでディレクトリを登録し、利用するサブネット、セルフサービス権限、アクセス制御を設定します
  5. ユーザーを選択し、WindowsまたはLinuxのバンドルを選択してWorkSpaceをプロビジョニングします
  6. クライアントアプリまたはWeb Accessで接続確認を行います
  7. 必要に応じて、カスタムイメージ、カスタムバンドル、Cost Optimizerを構成します
⚠️ 注意

WorkSpacesはディレクトリ、DNS、サブネット、セキュリティグループ、ルートテーブルの影響を強く受けます。WorkSpace単体のステータスだけでなく、Directory Service側のセキュリティグループやVPC DNS設定も合わせて確認します

Directory Service連携

WorkSpacesでは、AWS Managed Microsoft AD、Simple AD、AD Connectorなどのディレクトリを利用します。AD Connectorを使う場合は、ディレクトリに接続するためのサービスアカウントを用意し、必要最小限の権限を委任します

確認項目 確認内容
DNS WorkSpaces用VPCからディレクトリのDNS名を解決できること
ルーティング VPC PeeringやClient VPN経由でAD Connector、Domain Controllerへ到達できること
セキュリティグループ DNS、Kerberos、LDAPなど、AD ConnectorとDomain Controller間に必要な通信が許可されていること
サービスアカウント Domain Adminsではなく、AD Connector用の最小権限アカウントを作成して利用すること
ℹ️ 情報

AD Connectorのサービスアカウントには、ユーザーとグループの読み取り権限が必要です。Seamless Domain JoinやWorkSpacesを使う場合は、ドメイン参加やコンピューターオブジェクト作成に関する権限も確認します

公式情報: Getting started with AD Connector

WorkSpacesディレクトリ登録

Directory Serviceで作成または接続したディレクトリを、Amazon WorkSpaces側で登録します。登録後に、利用サブネット、アクセス制御、セルフサービス権限、Web Accessを設定します

項目 設定・確認観点
ディレクトリ WorkSpacesで利用するDirectory Serviceのディレクトリを選択します
サブネット 異なるAZのPrivate Subnetを選択します。WorkSpacesのネットワーク到達性とDNS解決を確認します
アクセス制御 接続元、クライアント種別、Web Accessの利用可否を確認します
ユーザー WorkSpaceは基本的に1ユーザーに対して1台を割り当てます。複数ユーザーで共有する前提にしないよう、要件を確認します

WorkSpaces起動

  1. Amazon WorkSpacesのナビゲーションから、WorkSpacesのプロビジョニングを開始します
  2. 登録済みディレクトリを選択します
  3. 対象ユーザーを選択します
  4. バンドル、実行モード、暗号化、ルートボリューム、ユーザーボリュームを確認します
  5. 作成後、ステータスがAvailableになることを確認します
  6. WorkSpacesクライアントまたはWeb Accessでサインインします
⚠️ 注意

カスタムイメージを作成する予定があるWorkSpaceでは、ボリューム暗号化の扱いに注意します。AWS公式ドキュメントでは、暗号化されたWorkSpaceからのイメージ作成はサポートされないとされています

Web Access

Web Accessを使うと、WorkSpacesクライアントアプリをインストールせずに、対応ブラウザからWorkSpacesへ接続できます。利用するには、管理者側でWeb Accessを有効化します

  1. Amazon WorkSpacesのディレクトリ設定を開きます
  2. ディレクトリの詳細更新から、アクセス制御オプションを確認します
  3. Web Accessを有効化します
  4. 必要に応じて、対象WorkSpaceをリビルドまたは再作成します
  5. WorkSpacesクライアントページ からWeb Accessに接続します
ℹ️ 情報

現在のWorkSpaces Web Accessは、DCVベースのWindows / Linux WorkSpacesで利用できます。PCoIPベースのWorkSpacesにはリージョンや機能の制約があります

公式情報: WorkSpaces Web Access

カスタムイメージ・バンドル

WorkSpaceにOS設定やアプリケーションを入れた後、同じ構成を再利用するためにカスタムイメージとカスタムバンドルを作成します

区分 確認内容
イメージ作成前 WorkSpaceのステータスがAvailableで、変更状態がNoneであることを確認します
Windows Image Checkerを実行し、失敗または警告を解消してからイメージ化します
Linux ユーザーボリューム配下にアプリケーションを含めないようにし、必要コンポーネントを確認します
暗号化 暗号化されたWorkSpaceからはカスタムイメージを作成できないため、要件定義時点で暗号化の必要性とイメージ運用を整理します
バンドル カスタムイメージとコンピュート・ストレージ構成を組み合わせ、以後のWorkSpaces作成時に利用します

公式情報: Create a custom WorkSpaces image and bundle for WorkSpaces Personal

Cost Optimizer

Cost Optimizer for Amazon WorkSpacesは、WorkSpacesの利用状況を分析し、利用実績に応じて時間課金または月額課金の適切なモードへ変換するAWS Solutionsです

  1. AWS SolutionsのCost OptimizerページからCloudFormationテンプレートを開きます
  2. 新規VPCに作成するか、既存VPCに配置するかを確認します
  3. WorkSpacesが存在するリージョンと、Cost Optimizerをデプロイするリージョンを確認します
  4. Dry Runの有無を確認します。初回はDry Runで結果を確認してから本適用する運用にします
  5. 出力レポート、ECSタスク、EventBridgeスケジュール、S3バケット、CloudWatch Logsを確認します
⚠️ 注意

Cost Optimizerのレポートが空になる場合、WorkSpacesの対象リージョン、CloudFormationパラメータ、実行タスクのログ、VPCエンドポイントの通信許可を確認します。VPC Interface Endpointを利用している環境では、ECSタスクやLambdaからWorkSpaces APIへ到達できるセキュリティグループ設定も確認します

公式情報: Cost Optimizer for Amazon WorkSpaces

トラブルシュート

事象 確認観点
WorkSpaceがエラーになる ディレクトリの登録状態、ユーザー割り当て、Domain ControllerのSecurity Group、VPC DNS設定、サブネットのルートを確認します
AD認証できない VPC CIDRを追加している場合、Directory Service側Security Groupの許可範囲が初期CIDRだけになっていないか確認します
Web Accessできない ディレクトリ設定でWeb Accessが有効か、対象WorkSpaceが対応プロトコルか、設定変更後にリビルドが必要か確認します
カスタムイメージが作成できない WorkSpace暗号化、Image Checker結果、ユーザープロファイル、Windows Updateの未完了、ドメインユーザーで動作するサービスを確認します
Cost OptimizerのExcelやレポートが空 対象リージョン、Dry Run設定、実行ログ、WorkSpaces API到達性、VPC EndpointのSecurity Groupを確認します

作業チェックリスト

タイミング 確認内容

構築中 Directory Service、WorkSpacesディレクトリ登録、サブネット、Security Group、ルートテーブルを確認します
接続確認 WorkSpacesクライアント、Web Access、AD認証、ユーザープロファイル、通信経路を確認します
運用開始前 カスタムイメージ要件、暗号化要件、Cost Optimizer、バックアップ・スナップショット、手順書を確認します
