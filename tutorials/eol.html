EOL / ライフサイクル一覧表

OS、AWSマネージドサービス、ミドルウェア、運用エージェントのサポート期限と主要な世代差分を整理した一覧表です

収録パッケージと導入元リポジトリの調査方法は、OSパッケージ一覧・導入元確認へ分離して整理しています

⚠️ 注意 EOLやサポート期限は変更される可能性があるため、作業前に公式ドキュメントを確認します

パッケージと導入元の確認

サポート期限を判断する前に、サーバーへ導入されているバージョンとリポジトリを確認します

OSと有効なリポジトリ

root shell
cat /etc/os-release
dnf repolist --enabled

パッケージのバージョンと導入元

root shell
PACKAGE=<PACKAGE_NAME>

rpm -q "$PACKAGE"
dnf list --installed "$PACKAGE"
dnf info "$PACKAGE"
dnf repoquery --installed --qf '%{name} %{version}-%{release} | %{arch} | %{repoid}' "$PACKAGE"
rpm -q --qf '%{NAME} %{VERSION}-%{RELEASE} | Vendor: %{VENDOR} | Packager: %{PACKAGER}\n' "$PACKAGE"
ℹ️ 情報 dnf list --installedのリポジトリ欄は@rhel-9-for-x86_64-appstream-rpmsや@remi-modularのように表示されます
dnf repoqueryを利用できない場合はdnf install -y dnf-plugins-coreで追加します
ミドルウェア別の実行例

以下は表示形式の例です
バージョン、リリース番号、アーキテクチャ、リポジトリIDは環境によって異なります

root shell
# Red Hat Application StreamsのPHP
dnf repoquery --installed --qf '%{name} %{version}-%{release} | %{repoid}' php
# php 8.0.x-x.el9 | rhel-9-for-x86_64-appstream-rpms

# Remi RepositoryのPHP
dnf repoquery --installed --qf '%{name} %{version}-%{release} | %{repoid}' php
# php 8.2.x-x.el9.remi | remi-modular

# EPELのhtop
dnf repoquery --installed --qf '%{name} %{version}-%{release} | %{repoid}' htop
# htop 3.x-x.el9 | epel

# PostgreSQL Global Development GroupのPostgreSQL
dnf repoquery --installed --qf '%{name} %{version}-%{release} | %{repoid}' postgresql16-server
# postgresql16-server 16.x-xPGDG.rhel9 | pgdg16

# Zabbix公式リポジトリのAgent 2
dnf repoquery --installed --qf '%{name} %{version}-%{release} | %{repoid}' zabbix-agent2
# zabbix-agent2 6.0.x-x.el9 | zabbix

# NGINX公式リポジトリ
dnf repoquery --installed --qf '%{name} %{version}-%{release} | %{repoid}' nginx
# nginx 1.x.x-x.el9.ngx | nginx-stable

Linux世代差分

対象 基盤・パッケージ管理 標準提供コンポーネント・運用
CentOS 7
  • RHEL 7互換のRPMベース
  • パッケージ管理: yum
  • 標準リポジトリ終了後はVaultへ移行
  • Software CollectionsとEPELを併用する構成あり
  • Python 2.7、OpenSSL 1.0.2
  • systemd、rsyslog、cronie
  • /var/log/messages中心のログ運用
  • ifconfig、netstatを使う旧手順が残存
Red Hat Enterprise Linux 8
  • FedoraをアップストリームとするRPMベース
  • BaseOSとAppStreamを分離
  • パッケージ管理: dnf、yumは互換コマンド
  • 複数バージョンはDNF moduleで切り替え
  • NetworkManagerによるネットワーク管理
  • 初期Application Stream: PHP 7.2、MariaDB 10.3、PostgreSQL 10、MySQL 8.0
  • Python 3.6、OpenSSL 1.1.1
  • systemd-journald、rsyslog、cronie
  • 追加ストリーム: PHP 7.3 / 7.4 / 8.0 / 8.2、MariaDB 10.5 / 10.11、PostgreSQL 12 / 13 / 15 / 16
Red Hat Enterprise Linux 9
  • FedoraをアップストリームとするRPMベース
  • BaseOSとAppStreamを分離
  • パッケージ管理: dnf
  • 初期バージョンは通常RPM、追加バージョンはDNF moduleを含む
  • NetworkManager、SELinux enforcing、システム全体のcrypto-policies
  • Full Life Application Stream: PHP 8.0、MariaDB 10.5、PostgreSQL 13、Apache HTTP Server 2.4、NGINX 1.20
  • Python 3.9、OpenSSL 3
  • systemd-journald、rsyslog、cronie
  • 追加ストリーム: PHP 8.1 / 8.2 / 8.3、MariaDB 10.11、PostgreSQL 15 / 16 / 18
Red Hat Enterprise Linux 10
  • FedoraをアップストリームとするRPMベース
  • BaseOSとAppStreamを分離
  • パッケージ管理: dnf
  • DNF modularityは非推奨で、Application Streamsをモジュール形式では配布しない
  • NetworkManager、SELinux enforcing、crypto-policies
  • Full Life Application Stream: PHP 8.3、MariaDB 10.11、PostgreSQL 16、Apache HTTP Server 2.4、NGINX 1.26、Valkey 8
  • Python 3.12、OpenSSL 3
  • systemd-journald、rsyslog、cronie
  • 追加ストリーム: PHP 8.4、MariaDB 11.8、MySQL 8.4、PostgreSQL 18
Amazon Linux 2
  • RHEL / Fedora系のRPMベース
  • パッケージ管理: yum
  • Amazon Linux Extrasで追加ランタイムを有効化
  • ローリング更新型の標準リポジトリ
  • Python 2.7 / 3系、OpenSSL 1.0.2
  • systemd、rsyslog、cronie
  • ExtrasからPHP、NGINX、MariaDB、PostgreSQLなどを選択
  • AL2023移行時はExtras廃止とパッケージ名変更を確認
Amazon Linux 2023
  • FedoraをアップストリームとするRPMベース
  • パッケージ管理: dnf
  • Amazon Linux Extrasを廃止
  • バージョン固定リポジトリとdnf upgrade --releasever=...
  • Python 3、OpenSSL 3、SELinux permissive
  • systemd-journaldを標準使用
  • rsyslogとcronieは標準AMIに未導入のため必要時に追加
  • 標準リポジトリでMariaDB系、PostgreSQL、Apache HTTP Server、NGINXなどを提供

PHP 8.0とPHP 8.2のサポート期限

  • Red Hat標準リポジトリから導入したパッケージは、PHPコミュニティのEOLではなくRed HatのApplication Streamライフサイクルに従う
  • Red Hat Enterprise Linux 9のPHP 8.0はFull Life Application Streamのため、Red Hat Enterprise Linux 9のライフサイクルに沿って修正がバックポートされる
  • PHP 8.2とPHP 8.3は個別期限を持つ追加ストリームで、Red Hat Enterprise Linux 9では2029/05に終了予定
  • BaseOSとAppStreamは別リポジトリで、Application StreamごとにOS本体より短い期限が設定される場合がある
  • dnf module list phpと公式ライフサイクル表で、選択中のストリームと期限をセットで確認する
