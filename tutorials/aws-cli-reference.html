AWS CLI コマンドリファレンス

日常運用でよく使うAWS CLIの基本確認、IAM、Security Group、Prefix List、SNSなどのコマンドを整理します。

📍 使い分け

本ページは、コンソールでも可能だがCLIの方が早い定型操作をまとめます。CLI/APIでしか変更・切り戻し・復旧できない操作は、AWS CLI 特殊対応 に分けています。

AWS CLIの初期設定

インストール

作業端末のOSに合わせてAWS CLIをインストールします。

powershell
winget install --id Amazon.AWSCLI -e --source winget
aws --version
zsh
brew install awscli
aws --version
bash
curl "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" -o "awscliv2.zip"
unzip awscliv2.zip
sudo ./aws/install
aws --version

認証情報の確認

現在の認証情報が、想定しているAWSアカウントとIAMユーザー/ロールを向いているか確認します。

powershell
aws sts get-caller-identity

認証情報の設定

作業端末で初めてAWS CLIを使う場合は、デフォルト認証情報を設定します。

powershell
aws configure
ℹ️ 情報 --profile は、複数のAWSアカウントやIAMユーザーを名前付きプロファイルで切り替える場合だけ指定します。このページでは、デフォルト認証情報を使う前提で記載します。

リージョン確認

作業前に対象リージョンを明示します。東京リージョンで作業する場合は ap-northeast-1 を指定します。

powershell
aws configure set region ap-northeast-1

AWS Identity and Access Management (IAM) / AWS IAM Identity Center

原則として人のログイン用途はAWS IAM Identity Centerを優先します。AWS Identity and Access Management (IAM) のIAMユーザーを作成する場合は、用途、権限、利用期限、アクセスキーの有無を事前に確認します。

値説明
<USER_NAME>作成または確認するIAMユーザー名
<GROUP_NAME>ユーザーを所属させるIAMグループ名

IAMユーザーを作成します。

powershell
aws iam create-user --user-name <USER_NAME>

作成したIAMユーザーを既存グループに追加します。

powershell
aws iam add-user-to-group --user-name <USER_NAME> --group-name <GROUP_NAME>

ユーザーの所属グループを確認します。

powershell
aws iam list-groups-for-user --user-name <USER_NAME>

Security Group ルール追加・削除

Security Group変更は影響範囲が広いため、対象SG、方向、プロトコル、ポート、送信元/宛先、説明欄を事前に確定します。

AWS CLIのSecurity Groupルールで全プロトコルを指定する場合は、公式リファレンスに合わせて --protocol -1 を使用します。

値説明
<SECURITY_GROUP_ID>変更対象のSecurity Group ID
<CIDR>許可または削除する送信元CIDR。例: 203.0.113.10/32

作業前に、対象Security Groupの現在のルールを確認します。

powershell
aws ec2 describe-security-groups --group-ids <SECURITY_GROUP_ID>

HTTPの80/tcpだけを許可する例です。

powershell
aws ec2 authorize-security-group-ingress --group-id <SECURITY_GROUP_ID> --protocol tcp --port 80 --cidr <CIDR>

全プロトコル・全ポートを許可する例です。検証や一時作業以外では原則として避け、作業後に必ず削除します。

powershell
aws ec2 authorize-security-group-ingress --group-id <SECURITY_GROUP_ID> --protocol -1 --cidr <CIDR>

80/tcpの許可ルールを削除する例です。

powershell
aws ec2 revoke-security-group-ingress --group-id <SECURITY_GROUP_ID> --protocol tcp --port 80 --cidr <CIDR>

全プロトコル・全ポートの許可ルールを削除する例です。

powershell
aws ec2 revoke-security-group-ingress --group-id <SECURITY_GROUP_ID> --protocol -1 --cidr <CIDR>

Prefix List作成・IP登録

接続元IPが複数ある場合は、Security GroupへCIDRを1件ずつ直接追加するのではなく、Customer-managed Prefix ListにまとめてからSecurity Groupで参照します。変更対象をPrefix List側に寄せられるため、接続元IPの追加・削除を運用しやすくなります。詳細は Managed prefix lists、create-managed-prefix-list、modify-managed-prefix-list、authorize-security-group-ingress を確認します。

⚠️ 注意 Security Groupの画面上は1つのPrefix List参照として整理できます。ただし、Customer-managed Prefix ListをSecurity Groupで参照する場合、クォータ上はPrefix Listの最大エントリ数分として扱われます。最大エントリ数は余裕を持たせすぎず、運用に必要な範囲で設計します。
値説明
<PREFIX_LIST_NAME>作成または更新するPrefix List名。
<MAX_ENTRIES>Prefix Listに登録できる最大エントリ数。例: 10
<CIDR_1>登録する接続元CIDR。例: 203.0.113.10/32
<CIDR_2>追加で登録する接続元CIDR。不要な場合はコマンドから削除します。
<CIDR_3>既存Prefix Listへ後から追加する接続元CIDR。
<DESCRIPTION_1>登録CIDRの説明。例: office-a
<DESCRIPTION_2>追加CIDRの説明。
<DESCRIPTION_3>後から追加するCIDRの説明。
<PREFIX_LIST_ID>作成後に払い出されるPrefix List ID。
<CURRENT_VERSION>Prefix List更新時に指定する現在のバージョン。
<SECURITY_GROUP_ID>Prefix Listを参照させるSecurity Group ID。
<PORT>許可するポート番号。例: 443

Prefix Listを作成し、初期CIDRを登録します。登録するIPアドレスが複数ある場合は、Cidr=... の指定を追加します。

powershell
aws ec2 create-managed-prefix-list --prefix-list-name <PREFIX_LIST_NAME> --address-family IPv4 --max-entries <MAX_ENTRIES> --entries Cidr=<CIDR_1>,Description="<DESCRIPTION_1>" Cidr=<CIDR_2>,Description="<DESCRIPTION_2>"

作成したPrefix ListのID、現在のバージョン、最大エントリ数を確認します。

powershell
aws ec2 describe-managed-prefix-lists --filters Name=prefix-list-name,Values=<PREFIX_LIST_NAME> --query "PrefixLists[*].[PrefixListId,PrefixListName,Version,MaxEntries,State]" --output table

既存のPrefix Listへ接続元CIDRを追加します。更新時は、直前に確認した Version を <CURRENT_VERSION> に指定します。

powershell
aws ec2 modify-managed-prefix-list --prefix-list-id <PREFIX_LIST_ID> --current-version <CURRENT_VERSION> --add-entries Cidr=<CIDR_3>,Description="<DESCRIPTION_3>"

Prefix Listに登録されているCIDRを確認します。

powershell
aws ec2 get-managed-prefix-list-entries --prefix-list-id <PREFIX_LIST_ID> --query "Entries[*].[Cidr,Description]" --output table

Security GroupのインバウンドルールでPrefix Listを参照します。例では、Prefix Listに登録した接続元CIDR群から <PORT> へのTCP通信を許可します。

powershell
aws ec2 authorize-security-group-ingress --group-id <SECURITY_GROUP_ID> --ip-permissions IpProtocol=tcp,FromPort=<PORT>,ToPort=<PORT>,PrefixListIds=[{PrefixListId=<PREFIX_LIST_ID>}]

SNS Topic・Subscription作成

通知先を作成する場合は、Topic、Subscription、確認メールの承認、通知テストまでを一連の手順として扱います。

SNS Topicを作成します。

powershell
aws sns create-topic --name <TOPIC_NAME>

メール通知先をSubscriptionとして登録します。

powershell
aws sns subscribe --topic-arn <TOPIC_ARN> --protocol email --notification-endpoint <MAIL_ADDRESS>

通知テストを実行します。

powershell
aws sns publish --topic-arn <TOPIC_ARN> --message "notification test"
