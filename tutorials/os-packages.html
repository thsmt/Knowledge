OSパッケージ一覧・導入元確認

ℹ️ 情報 公開パッケージ一覧はOSで提供可能なパッケージを示します。実サーバーの導入バージョン、リポジトリ、モジュールストリームはコマンド結果を基準に確認します

1. OSとリポジトリ

1.1 OSリリース

root shell
cat /etc/os-release
uname -r
rpm -q rpm dnf

1.2 有効なリポジトリ

root shell
dnf repolist --enabled
dnf repolist --all

2. 導入済みパッケージ

2.1 バージョンとRPM情報

パッケージ名、バージョン、リリース、アーキテクチャを確認します

root shell
rpm -q httpd php mariadb-server postgresql-server openssl rsyslog
rpm -qi httpd

実行例

console
$ rpm -q httpd openssl rsyslog
httpd-2.4.62-1.el9_5.2.x86_64
openssl-3.2.2-6.el9_5.1.x86_64
rsyslog-8.2310.0-4.el9.x86_64

2.2 導入元リポジトリ

導入済みパッケージのリポジトリIDを確認します

root shell
dnf info installed httpd
dnf repoquery --installed --qf '%{name} %{epoch}:%{version}-%{release}.%{arch} %{repoid}' \
  httpd php mariadb-server postgresql-server openssl rsyslog

実行例

console
$ dnf repoquery --installed --qf '%{name} %{version}-%{release} %{repoid}' httpd
httpd 2.4.62-1.el9_5.2 rhel-9-appstream-rhui-rpms
⚠️ 注意 @Systemと表示される場合はRPMデータベース上の導入済み情報だけを示し、現在の取得元を特定できないことがあります。dnf info installed、有効リポジトリ、導入履歴を併せて確認します

2.3 ファイルを提供するパッケージ

root shell
rpm -qf /usr/sbin/httpd
rpm -qf /usr/bin/openssl
dnf provides '*/semanage'

2.4 導入・更新履歴

root shell
dnf history list
dnf history info <TRANSACTION_ID>

3. RHEL Application Streams

モジュールストリームを利用するパッケージは、有効なストリームとプロファイルを確認します

root shell
dnf module list
dnf module list php
dnf module list postgresql
dnf module info php:8.2
ℹ️ 情報 RHELのApplication Streamは、同じRHELメジャーリリース内でもストリームごとにライフサイクルが異なります。導入元がBaseOS、AppStream、EPEL、Remiなどのどれかを確認し、その提供元の期限を参照します
