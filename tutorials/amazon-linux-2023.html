[構築] Amazon Linux 2023

ℹ️ 前提 実環境名、IPアドレス、顧客固有値は扱わず、再利用できる手順として一般化しています。<HOSTNAME>、<FQDN>、<PRIVATE_IP>、<RELEASE_VERSION> などは環境に合わせて置き換えます

0. 起動時の確認

本手順では、適用先の組織ポリシーで指定された ED25519 キーペアを使用します。Amazon EC2はRSAキーペアにも対応していますが、鍵方式や鍵長が組織のセキュリティ基準に適合しないRSA鍵は使用しません

確認項目 内容
キーペアED25519 を利用します
接続方式SSH、Session Manager、踏み台経由など、作業経路を事前に確認します
バックアップ設定変更前にAMI、EBS Snapshot、設定ファイル退避の要否を確認します

2. SELinuxの状態確認

Amazon Linux 2023ではSELinuxが初期状態で Permissive です。Red Hat Enterprise Linux 9のように permissive へ変更する手順は標準では不要です

2.1 SELinuxの動作モード確認

root shell
getenforce

2.2 SELinux設定ファイルの確認

root shell
grep '^SELINUX=' /etc/selinux/config

4. ログインプロファイル設定

export EDITOR=vi をシェルで直接実行しただけでは、現在のシェルと子プロセスにしか反映されず、再起動後には残りません。ログイン後も継続して利用する設定は /etc/profile.d/ に配置します

4.1 履歴設定ファイルの作成

root shell
cat > /etc/profile.d/history.sh <<'EOF'
export HISTSIZE=10000
export HISTTIMEFORMAT='%F %T '
EOF
chmod 644 /etc/profile.d/history.sh

4.2 デフォルトエディタ設定ファイルの作成

root shell
cat > /etc/profile.d/editor.sh <<'EOF'
export EDITOR=vi
export VISUAL=vi
EOF
chmod 644 /etc/profile.d/editor.sh

4.3 設定の反映と確認

root shell
source /etc/profile.d/history.sh
source /etc/profile.d/editor.sh
printf 'EDITOR=%s\nVISUAL=%s\n' "$EDITOR" "$VISUAL"

7. sshd_config修正

SSHのrootログイン、タイムアウト、DNS逆引き、SFTPログ出力を確認・調整します。キーペアは起動時に ED25519 で作成する前提です

7.1 sshd_configのバックアップ取得

root shell
cp -pi /etc/ssh/sshd_config /etc/ssh/sshd_config.org

7.2 sshd_configの編集

root shell
vi /etc/ssh/sshd_config

7.3 SSH設定値の確認

sshd_config
PermitRootLogin no
UseDNS no
ClientAliveInterval 300
ClientAliveCountMax 2
Subsystem sftp /usr/libexec/openssh/sftp-server -f LOCAL3 -l VERBOSE

7.4 構文確認とsshd再起動

root shell
sshd -t
systemctl restart sshd
systemctl status sshd

9. 追加ディスクのマウント

追加EBSのデバイス名は、EC2の指定値とOS上の表示が異なることがあります。Nitroベースのインスタンスでは、通常 /dev/nvme1n1 のようなNVMeデバイスとして認識されます

9.1 デバイスとファイルシステムの確認

root shell
lsblk -f
file -s /dev/<DEVICE_NAME>
⚠️ 注意 file -s の結果が data の場合は、ファイルシステムがありません。既存のファイルシステムやデータがあるボリュームに mkfs を実行するとデータが失われるため、対象デバイスを必ず確認します

9.2 ファイルシステムの作成

root shell
mkfs -t xfs /dev/<DEVICE_NAME>

9.3 マウントポイントの作成とマウント

root shell
mkdir -p <MOUNT_POINT>
mount /dev/<DEVICE_NAME> <MOUNT_POINT>
df -hT <MOUNT_POINT>

9.4 UUIDの確認

root shell
blkid /dev/<DEVICE_NAME>

9.5 fstabへの永続化

root shell
cp -pi /etc/fstab /etc/fstab.orig
vi /etc/fstab
fstab
UUID=<UUID> <MOUNT_POINT> xfs defaults,nofail 0 2

9.6 fstab設定の確認

root shell
umount <MOUNT_POINT>
mount -a
findmnt <MOUNT_POINT>
df -hT <MOUNT_POINT>

10. Amazon EFSのマウント

Amazon EFSを利用する場合は、EFS用Security Group、マウントターゲット、マウント方式を確認します。不要な場合はこの手順をスキップします

ℹ️ 情報

amazon-efs-utils はAmazon Linux 2023のリポジトリから導入できますが、すべてのAMIにインストール済みとは限りません。パッケージの有無を確認し、未導入の場合にインストールします

通常は、転送中の暗号化を利用できるEFSマウントヘルパーを使用します。IPアドレスを指定したNFSv4.1マウントは、疎通確認やマウントターゲットを明示する要件がある場合に利用します

10.1 EFSマウントヘルパーの確認と導入

root shell
rpm -q amazon-efs-utils || dnf install -y amazon-efs-utils

EFS用Security Groupでは、対象EC2のSecurity GroupまたはプライベートIPから 2049/tcp を許可します

10.2 マウントポイントの作成とTLSマウント

root shell
mkdir -p /mnt/efs
mount -t efs -o tls <EFS_FILE_SYSTEM_ID>:/ /mnt/efs
df -Th

10.3 マウントターゲットIPを指定する場合

root shell
mount -t nfs4 -o nfsvers=4.1,rsize=1048576,wsize=1048576,hard,timeo=600,retrans=2,noresvport <EFS_MOUNT_TARGET_IP>:/ /mnt/efs/
df -Th

10.4 fstabへの永続化

root shell
cp -pi /etc/fstab /etc/fstab.orig
vi /etc/fstab
fstab
<EFS_FILE_SYSTEM_ID>:/ /mnt/efs efs defaults,_netdev,tls 0 0

10.5 fstab設定の確認

root shell
mount -a
df -Th

11. swap領域作成

11.1 swap領域の確認

root shell
free | grep Swap | awk '{print $2}'

11.2 空き容量の確認

root shell
df -h | grep /$ | awk '{print $4}'

11.3 swapファイルの作成

2GiBのswapファイルを作成し、rootだけが読み書きできる権限を設定します

root shell
dd if=/dev/zero of=/.swapfile bs=512M count=4
chmod 600 /.swapfile

11.4 swapの有効化

root shell
mkswap /.swapfile
swapon /.swapfile

11.5 swap領域の再確認

root shell
free | grep Swap | awk '{print $2}'
swapon --show

11.6 fstabへの永続化

root shell
cp -pi /etc/fstab /etc/fstab.orig
vi /etc/fstab
fstab
/.swapfile    swap    swap    defaults    0    0

11.7 fstab設定の確認

root shell
mount -a

12. ミドルウェアの導入

12.1 提供パッケージの確認

root shell
dnf search httpd
dnf search mariadb
dnf search postgresql

12.2 Web・メール・DBクライアントの導入

MySQL互換のクライアント用途では、Amazon Linux 2023の標準リポジトリ上にあるMariaDB系パッケージを確認して導入します。PostgreSQLも同様に、利用するメジャーバージョンを確認してから導入します

root shell
dnf install -y httpd httpd-devel httpd-tools mod_ssl
dnf install -y postfix cyrus-sasl-plain s-nail
dnf install -y mariadb105
dnf install -y postgresql15
ℹ️ 情報

外部リポジトリを追加する前に、標準リポジトリで提供されるパッケージとサポート方針を確認します

DBサーバー本体を構築する場合は、mariadb105-server や postgresql15-server など、環境で利用するバージョンに合わせて別途検証します

16. Amazon Linux 2023リリース更新

Amazon Linux 2023はリリースバージョンを指定して更新できます。新しいリリースが利用可能な場合は、dnf check-release-update の結果を確認してから、適用するリリースを明示します

16.1 利用可能なリリースの確認

root shell
dnf check-release-update

16.2 出力例

example output
WARNING:
  A newer release of "Amazon Linux" is available.

Available Versions:
Version 2023.8.20250721:
  Run the following command to upgrade to 2023.8.20250721:
    dnf upgrade --releasever=2023.8.20250721
Version 2023.8.20250804:
  Run the following command to upgrade to 2023.8.20250804:
    dnf upgrade --releasever=2023.8.20250804

16.3 リリースバージョンを指定した更新

root shell
dnf upgrade --releasever=<RELEASE_VERSION>

16.4 再起動

root shell
reboot

16.5 更新後のOS情報確認

root shell
cat /etc/os-release

17. rsyslog設定

Amazon Linux 2023は systemd-journald が標準です。従来の /var/log/messages、/var/log/secure、/var/log/cron、SFTPログファイルを使う運用では rsyslog を追加導入します

17.1 rsyslogのインストールと起動

root shell
dnf install -y rsyslog
systemctl enable --now rsyslog
systemctl status rsyslog

17.2 SFTPログファイルの作成

SFTPログの出力先ファイルを作成し、所有者と権限を設定します

root shell
touch /var/log/sftp.log
chown root:root /var/log/sftp.log
chmod 600 /var/log/sftp.log

17.3 SFTPログ出力設定の作成

root shell
vi /etc/rsyslog.d/sftp.conf
/etc/rsyslog.d/sftp.conf
local3.*    /var/log/sftp.log

17.4 設定反映とログ出力確認

root shell
systemctl restart rsyslog
logger -p local3.info "sftp log test"
tail -n 20 /var/log/sftp.log

18. cron設定

Amazon Linux 2023ではcronがデフォルトでは利用できません。既存運用で crontab を使用する場合は cronie を導入し、crond を有効化します

18.1 cronieのインストール

root shell
dnf install -y cronie cronie-noanacron

18.2 crondの起動

root shell
systemctl enable --now crond
systemctl status crond

18.3 rootのcrontab確認

root shell
crontab -l

19. sudoers編集時のエディタ指定

デフォルトエディタは、4.2で作成した /etc/profile.d/editor.sh によりログイン時に設定されます。現在のシェルへ未反映の場合は、設定を読み込んでから visudo を実行します

19.1 エディタ設定の反映とsudoers編集

root shell
source /etc/profile.d/editor.sh
visudo

19.2 sudoersの構文確認

root shell
visudo -c

20. logrotate設定

rsyslogやミドルウェアのログをファイル出力する場合は、logrotate設定を確認します

20.1 共通設定のバックアップ取得と編集

root shell
cp -pi /etc/logrotate.conf /etc/logrotate.conf.orig
vi /etc/logrotate.conf

20.2 個別設定ファイルの確認

root shell
ls -l /etc/logrotate.d/
cat /etc/logrotate.d/syslog

20.3 SFTPログのローテーション設定

root shell
vi /etc/logrotate.d/sftp
/etc/logrotate.d/sftp
/var/log/sftp.log {
    weekly
    rotate 12
    missingok
    notifempty
    compress
    create 0600 root root
}

20.4 設定内容の検証

root shell
logrotate -d /etc/logrotate.conf
