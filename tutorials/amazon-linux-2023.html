[構築] Amazon Linux 2023

AWS EC2上のAmazon Linux 2023を手動構築する際の初期設定を、[構築] Red Hat Enterprise Linux 9 の順番に合わせて整理します。Red Hat Enterprise Linux 9と同じ考え方で扱う項目、Amazon Linux 2023では不要な項目、追加導入が必要になりやすい項目を分けて記載します。root権限で実行する前提のため、コマンド例では sudo を省略しています。

ℹ️ 前提 実環境名、IPアドレス、顧客固有値は扱わず、再利用できる手順として一般化しています。<HOSTNAME>、<FQDN>、<PRIVATE_IP>、<RELEASE_VERSION> などは環境に合わせて置き換えます。

1. 起動時の確認

AWSコンソールでEC2を起動する場合、キーペアは ED25519 で作成します。RSA鍵は組織ポリシーや鍵長の条件で拒否される場合があります。

確認項目 内容
キーペアED25519 を利用します。
接続方式SSH、Session Manager、踏み台経由など、作業経路を事前に確認します。
バックアップ設定変更前にAMI、EBS Snapshot、設定ファイル退避の要否を確認します。

2. SELinuxの状態確認

Amazon Linux 2023ではSELinuxが初期状態で Permissive です。Red Hat Enterprise Linux 9のように permissive へ変更する手順は標準では不要です。

SELinuxの状態を確認します。

root shell
getenforce

設定ファイルも確認します。

root shell
grep '^SELINUX=' /etc/selinux/config
ℹ️ 補足 Amazon Linux 2023で Enforcing を利用する場合は、対象ミドルウェア、ログ出力、運用手順を別途検証します。このページでは標準状態の確認に留めます。

3. subscription-managerの確認

Amazon Linux 2023ではRed Hat Subscription Managementを利用しないため、Red Hat Enterprise Linux 9のような subscription-manager 無効化は不要です。

パッケージが入っていないことを確認します。

root shell
rpm -q subscription-manager

利用中のリポジトリを確認します。

root shell
dnf repolist

4. ログインプロファイル設定

ログイン時の共通環境変数や履歴設定を整理するため、/etc/profile を退避します。

root shell
cp -pi /etc/profile /etc/profile.org

設定ファイルを編集します。

root shell
vi /etc/profile

必要に応じて追記します。

profile
export HISTSIZE=10000
export HISTTIMEFORMAT='%F %T '
export EDITOR=vi
export VISUAL=vi

差分を確認します。

root shell
diff -u /etc/profile.org /etc/profile

5. ホスト名とhosts設定

必要に応じてホスト名を変更します。

root shell
hostnamectl set-hostname <HOSTNAME>

ホスト名を確認します。

root shell
hostnamectl
hostname -f

/etc/hosts を退避して編集します。

root shell
cp -pi /etc/hosts /etc/hosts.org
vi /etc/hosts

プライベートIP、FQDN、ホスト名を追記します。

hosts
<PRIVATE_IP> <FQDN> <HOSTNAME>

6. タイムゾーン設定

サーバーのタイムゾーンを日本時間に変更します。

root shell
timedatectl set-timezone Asia/Tokyo

設定後の時刻を確認します。

root shell
timedatectl
date

7. sshd_config修正

SSHのrootログイン、タイムアウト、DNS逆引き、SFTPログ出力を確認・調整します。キーペアは起動時に ED25519 で作成する前提です。

設定ファイルを退避します。

root shell
cp -pi /etc/ssh/sshd_config /etc/ssh/sshd_config.org

設定ファイルを編集します。

root shell
vi /etc/ssh/sshd_config

必要に応じて、以下のような設定を確認します。

sshd_config
PermitRootLogin no
UseDNS no
ClientAliveInterval 300
ClientAliveCountMax 2
Subsystem sftp /usr/libexec/openssh/sftp-server -f LOCAL3 -l VERBOSE

構文確認後、sshdを再起動します。

root shell
sshd -t
systemctl restart sshd
systemctl status sshd

8. cloud-init設定変更

Amazon Linux 2023では、標準的な手動構築ではcloud-init設定を変更しません。必要な場合のみ、起動時ユーザーデータ、ホスト名管理、追加ディスクの自動マウントなどの要件を確認して個別に調整します。

確認項目 Amazon Linux 2023での扱い
ホスト名hostnamectl と /etc/hosts で設定します。
追加ディスク手動でマウント、または別途UserDataやCloudFormationで管理します。
root無効化AMI既定値とSSH設定を確認し、必要時のみ変更します。

9. 追加ディスクのマウント

追加EBSを利用する場合は、デバイス名、ファイルシステム、マウントポイント、/etc/fstab の永続化設定を確認します。不要な場合はこの手順をスキップします。

ディスク構成を確認します。

root shell
lsblk
df -hT

10. swap領域作成

現在のswap領域を確認します。

root shell
free | grep Swap | awk '{print $2}'

ルートファイルシステムの空き容量を確認します。

root shell
df -h | grep /$ | awk '{print $4}'

swapファイルを作成し、権限を変更します。Red Hat Enterprise Linux 9手順と同じ形式に合わせ、/.swapfile を利用します。

ℹ️ 補足 fallocate は短時間でswapファイルを作成できるため、検証用途では便利です。一方で、ファイルシステムや作成方式によってはswapファイルとして扱う際に差が出ることがあります。手順書としては環境差を減らし、Red Hat Enterprise Linux 9の手順とも揃えるため、実データを書き込む dd 方式に統一しています。
root shell
dd if=/dev/zero of=/.swapfile bs=512M count=4
chmod 600 /.swapfile

swapを有効化します。

root shell
mkswap /.swapfile
swapon /.swapfile

swap領域を再確認します。

root shell
free | grep Swap | awk '{print $2}'
swapon --show

再起動後も有効化されるように、/etc/fstab へ追記します。

root shell
cp -pi /etc/fstab /etc/fstab.org
vi /etc/fstab
fstab
/.swapfile    swap    swap    defaults    0    0

fstabの記載ミスがないか確認します。

root shell
mount -a

11. ミドルウェアのインストール方針

Amazon Linux 2023では dnf を利用します。yum も利用できますが、実体は dnf へ向くため、このページでは dnf で統一します。

利用可能なパッケージを確認します。

root shell
dnf search httpd
dnf search mariadb
dnf search postgresql

Webサーバー、メール関連、DBクライアントを導入する例です。MySQL互換のクライアント用途では、Amazon Linux 2023の標準リポジトリ上にあるMariaDB系パッケージを確認して導入します。PostgreSQLも同様に、利用するメジャーバージョンを確認してから導入します。

root shell
dnf install -y httpd httpd-devel httpd-tools mod_ssl
dnf install -y postfix cyrus-sasl-plain s-nail
dnf install -y mariadb105
dnf install -y postgresql15
ℹ️ 補足 外部リポジトリを追加する前に、標準リポジトリで提供されるパッケージとサポート方針を確認します。DBサーバー本体を構築する場合は、mariadb105-server や postgresql15-server など、環境で利用するバージョンに合わせて別途検証します。

12. Postfix設定

Postfixを利用する場合は、設定ファイルを退避してからホスト名やドメイン、リレー先などを設定します。

root shell
cp -pi /etc/postfix/main.cf /etc/postfix/main.cf.org
vi /etc/postfix/main.cf

設定後、サービスを有効化します。

root shell
postfix check
systemctl enable --now postfix
systemctl status postfix

13. Apache設定

Apacheを利用する場合は、設定ファイルを退避してから変更します。

root shell
cp -pi /etc/httpd/conf/httpd.conf /etc/httpd/conf/httpd.conf.org
vi /etc/httpd/conf/httpd.conf

構文確認後、サービスを有効化します。

root shell
apachectl configtest
systemctl enable --now httpd
systemctl status httpd

14. 基本コマンドのインストール

運用調査で利用する基本コマンドを導入します。

root shell
dnf install -y vim-enhanced telnet lftp mlocate libyaml wget mtr tcpdump tmux \
bash-completion net-tools bind-utils sysstat lsof tar zip unzip logrotate

導入後、主要コマンドを確認します。

root shell
rpm -q logrotate sysstat bind-utils net-tools

15. Amazon Linux 2023リリース更新

Amazon Linux 2023はリリースバージョンを指定して更新できます。新しいリリースが利用可能な場合は、dnf check-release-update の結果を確認してから、適用するリリースを明示します。

利用可能なリリースを確認します。

root shell
dnf check-release-update

実行例では、以下のように新しいAmazon Linuxのリリースが表示されます。

example output
WARNING:
  A newer release of "Amazon Linux" is available.

Available Versions:
Version 2023.8.20250721:
  Run the following command to upgrade to 2023.8.20250721:
    dnf upgrade --releasever=2023.8.20250721
Version 2023.8.20250804:
  Run the following command to upgrade to 2023.8.20250804:
    dnf upgrade --releasever=2023.8.20250804

適用するリリースを指定して更新します。

root shell
dnf upgrade --releasever=<RELEASE_VERSION>

必要に応じて再起動します。

root shell
reboot

再接続後、OS情報を確認します。

root shell
cat /etc/os-release

16. rsyslog設定

Amazon Linux 2023は systemd-journald が標準です。従来の /var/log/messages、/var/log/secure、/var/log/cron、SFTPログファイルを使う運用では rsyslog を追加導入します。

rsyslogをインストールして有効化します。

root shell
dnf install -y rsyslog
systemctl enable --now rsyslog
systemctl status rsyslog

SFTPログ出力用のファイルを事前に作成します。ファイルが存在しない状態で出力先だけを指定すると、環境によってログ出力や検証時にエラーとなる場合があります。

root shell
touch /var/log/sftp.log
chown root:root /var/log/sftp.log
chmod 600 /var/log/sftp.log

SFTPログを /var/log/sftp.log に出力する設定を作成します。

root shell
vi /etc/rsyslog.d/sftp.conf
/etc/rsyslog.d/sftp.conf
local3.*    /var/log/sftp.log

rsyslogを再起動し、テストログを書き込みます。

root shell
systemctl restart rsyslog
logger -p local3.info "sftp log test"
tail -n 20 /var/log/sftp.log

17. cron設定

Amazon Linux 2023ではcronがデフォルトでは利用できません。既存運用で crontab を使用する場合は cronie を導入し、crond を有効化します。

cronieをインストールします。

root shell
dnf install -y cronie cronie-noanacron

crondを有効化します。

root shell
systemctl enable --now crond
systemctl status crond

rootのcrontabを確認します。

root shell
crontab -l

18. sudoers編集時のエディタ指定

visudo 実行時に意図せず nano が開く場合は、作業セッションで利用するエディタを明示します。

root shell
export EDITOR=vi
export VISUAL=vi
visudo

編集後、構文を確認します。

root shell
visudo -c

19. logrotate設定

rsyslogやミドルウェアのログをファイル出力する場合は、logrotate設定を確認します。

共通設定を退避して編集します。

root shell
cp -pi /etc/logrotate.conf /etc/logrotate.conf.org
vi /etc/logrotate.conf

個別設定ファイルを確認します。

root shell
ls -l /etc/logrotate.d/
cat /etc/logrotate.d/syslog

SFTPログをローテート対象にする場合は、個別設定を作成します。

root shell
vi /etc/logrotate.d/sftp
/etc/logrotate.d/sftp
/var/log/sftp.log {
    weekly
    rotate 12
    missingok
    notifempty
    compress
    create 0600 root root
}

logrotateのデバッグ実行で設定を確認します。

root shell
logrotate -d /etc/logrotate.conf
