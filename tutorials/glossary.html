RHEL・リポジトリ

用語意味
BaseOS主にRed Hat Enterprise Linux 8以降で採用される、起動、カーネル、systemd、シェル、ネットワークなどOSの動作に不可欠なコアパッケージ群
AppStream言語ランタイム、データベース、Webサーバーなどのユーザー空間アプリケーションを提供するRHELのリポジトリ。BaseOSと共にRHELの主要構成を形成
Application StreamRHELのライフサイクルから一部独立して、特定バージョンのアプリケーションを提供する仕組み。RPMまたはModuleとして配布
Module StreamPHPやPostgreSQLなど同一ソフトウェアの複数メジャーバージョンから利用系列を選択する仕組み
Module ProfileModule Stream内で導入するパッケージ構成を用途別にまとめたセット。common、server、clientなどを選択
CodeReady Linux Builder開発用ライブラリ、ヘッダー、ビルド依存などを提供するRHELの追加リポジトリ。パッケージごとのサポート範囲を確認して利用
RHUIRed Hat Update Infrastructure。AWSなどのクラウド環境でRHEL更新コンテンツを配信する仕組み
subscription-managerRHELシステムのサブスクリプション登録、コンテンツアクセス、リポジトリ有効化を管理するCLI
Content SetRed Hat CDNやSatelliteで配信されるリポジトリコンテンツの論理単位
Repository IDDNFがパッケージ取得元を識別する一意のリポジトリ名
EPELFedora ProjectがRHEL互換環境向けに提供する追加パッケージリポジトリ。RHEL本体とはサポート元と期限が異なる
Remi RepositoryRHEL互換環境向けに複数のPHP系列と関連パッケージを提供する外部リポジトリ
ErrataRed Hatが公開する更新情報の総称。セキュリティ、バグ修正、機能拡張を分類して提供
RHSARed Hat Security Advisory。脆弱性修正を含むセキュリティアドバイザリ
RHBARed Hat Bug Advisory。製品不具合の修正を提供するアドバイザリ
RHEARed Hat Enhancement Advisory。機能追加や改善を提供するアドバイザリ
EUSExtended Update Support。特定のRHELマイナーリリースへ長期間更新を提供する有償ライフサイクルオプション
ELSExtended Life Cycle Support。通常のMaintenance Support終了後も限定的な重大修正を提供する有償延長サポート
SatelliteRHELのコンテンツ、パッチ、サブスクリプション、プロビジョニングを組織内で集中管理する製品
Red Hat Insights構成とリスク情報を解析し、脆弱性、設定不備、推奨修正を提示するRed Hatの管理サービス
LeappRHELのインプレースメジャーアップグレードで事前評価と変換処理を行うフレームワーク
Kickstartパーティション、パッケージ、ユーザー、初期設定を定義してRHEL系OSを自動インストールする仕組み
AnacondaFedoraとRHEL系OSで使用されるインストーラー。対話インストールとKickstartを処理

RPM・依存関係

用語意味
RPMRHEL系OSでパッケージのファイル、依存関係、スクリプト、メタデータを管理する形式とコマンド
SRPMソースコード、パッチ、specファイルを収録し、バイナリRPMを再ビルドするためのソースRPM
specファイルRPMの名称、バージョン、依存関係、ビルド手順、インストール内容、変更履歴を定義するファイル
NEVRAName、Epoch、Version、Release、ArchitectureでRPMを一意に識別する表記
Epoch通常のバージョン比較で新旧を表せない場合に優先順位を補正するRPMの数値
DNFリポジトリメタデータと依存関係を解決し、RPMの検索、導入、更新、削除、履歴管理を行うパッケージマネージャー
YUM従来のRHEL系パッケージマネージャー。RHEL 8以降では互換コマンドとしてDNFへ処理を委譲
GPG署名パッケージとリポジトリメタデータが信頼する配布元から提供され、改ざんされていないことを検証する電子署名
依存関係対象パッケージの動作や導入に必要な別パッケージ、ライブラリ、機能の関係
Weak Dependency必須ではないが追加機能や推奨構成に利用するRecommends、Suggestsなどの依存関係
ABIバイナリ間で呼び出し規約、データ配置、シンボルなどの互換性を定める境界
kABIカーネルと外部カーネルモジュール間のバイナリ互換性。ドライバーやセキュリティエージェントの継続動作に影響
Backport新しいバージョンで実装された脆弱性・不具合修正を、製品バージョンを大きく変えず既存系列へ取り込む保守方法

Linux基盤・ストレージ

用語意味
SELinuxプロセスとファイルへセキュリティコンテキストを割り当て、ポリシーに基づいてアクセスを制御する強制アクセス制御機構
Type EnforcementSELinuxでプロセスのドメインとファイルのタイプの組み合わせから許可される操作を判定する主要な制御方式
Security ContextSELinuxがユーザー、ロール、タイプ、レベルを組み合わせてプロセスやファイルへ付与するラベル
AVCAccess Vector Cache。SELinuxのアクセス判定をキャッシュし、拒否内容を監査ログへ記録する仕組み
SELinux Booleanポリシー全体を書き換えず、用意された機能単位の許可を有効または無効にする設定値
systemdサービス、ソケット、マウント、タイマーなどをUnitとして管理し、起動順序と依存関係を制御するシステムマネージャー
Unitsystemdが管理するservice、socket、target、mount、timerなどの設定単位
Target複数のUnitをまとめ、起動時の到達状態や依存関係を表すsystemdのUnit種別
journaldsystemdが提供する構造化ログ収集サービス。カーネル、サービス、標準出力のログをJournalへ保存
cgroupプロセス群のCPU、メモリ、I/Oなどの利用量を制限、計測、分離するLinuxカーネル機能
NamespaceプロセスID、ネットワーク、マウントポイント、ユーザーなどのシステム資源をプロセス群ごとに分離するLinuxカーネル機能
initramfsルートファイルシステムをマウントする前に必要なドライバーと初期化処理を格納する一時的なルートファイルシステム
dracut現在のカーネル、ストレージ、ネットワーク構成に合わせてinitramfsを生成するRHEL系のツール
kdump障害発生時に予約済みカーネルへ切り替え、稼働中カーネルのメモリ内容をvmcoreとして保存するクラッシュダンプ機構
kexecファームウェア初期化を経由せず、実行中のカーネルから別のカーネルを直接起動する仕組み。kdumpの切り替えに利用
tuned用途別プロファイルに基づいてCPU、I/O、電源管理、カーネルパラメータを調整するRHEL系の性能チューニングサービス
chronyNTPを使用してシステム時刻を同期し、断続的なネットワークや仮想環境にも対応する時刻同期実装
NetworkManagerネットワークデバイス、接続プロファイル、IPアドレス、DNS、ルーティングを一元管理するサービス
nmcliNetworkManagerの接続状態確認と設定変更を行うコマンドラインインターフェイス
firewalldZoneとServiceを用いてnftablesまたはiptablesのルールを動的に管理するホストファイアウォールサービス
LVM物理ディスクをPV、VG、LVの階層で抽象化し、容量拡張やスナップショットを柔軟に扱う論理ボリューム管理機構
PV / VG / LVPhysical Volume、Volume Group、Logical Volume。LVMで物理領域を束ね、利用可能な論理領域へ分割する構成要素
XFSRHELの既定ファイルシステム。オンライン拡張に対応する一方、ファイルシステムの縮小には対応しない
GRUB2起動対象のカーネルとカーネルパラメータを選択し、OS起動処理へ制御を渡すブートローダー

Amazon Linux

用語意味
Versioned RepositoryAmazon Linux 2023が特定リリースのリポジトリへ固定し、更新内容を予測可能にする仕組み。releaseverで更新先を指定
Deterministic Upgrade利用するAmazon Linux 2023リリースを明示し、同じ入力から同じパッケージ集合へ更新する考え方
SPALEPEL 9を基にAmazon Linux 2023向け追加パッケージを専用リポジトリで提供する仕組み
Supplemental Packages for Amazon Linux
AL2023 AMI特定のAmazon Linux 2023リリースとカーネル、初期パッケージを組み合わせたEC2用マシンイメージ

ソフトウェア系譜・配布

用語意味
Upstream機能開発と原典のソースコード管理を行う上流プロジェクト。Linux Kernel、Fedora、Apache HTTP Serverなど
DownstreamUpstreamを基に検証、パッチ、サポート、配布方針を加えた下流製品やディストリビューション
Fork既存プロジェクトのソースコードを分岐し、独立した方針と履歴で開発を継続すること
Vanillaベンダー固有パッチや追加変更を加えていないUpstreamそのままの状態を指す通称
DistributionLinuxカーネル、ユーザーランド、パッケージ管理、インストーラー、更新サービスを組み合わせて配布するOS製品
Derivative Distribution別のディストリビューションを基盤に、パッケージや設定、ブランド、サポート方針を変更した派生OS
Vendor PatchOSまたはソフトウェアベンダーがUpstream版へ追加するセキュリティ、互換性、サポート用の変更
LTSLong Term Support。通常版より長い期間、修正やサポートを提供するリリース方針
EOL / EOS製品・バージョンの保守またはサポートが終了する時点
CVE公開された脆弱性を一意に識別する共通番号
SBOMSoftware Bill of Materials。製品に含まれるソフトウェア、ライブラリ、バージョン、依存関係を一覧化した構成表
Provenance成果物がどのソース、ビルド環境、依存関係、工程から生成されたかを追跡する由来情報
