AWS CLI 実務手順集

AWS CLIを使った認証確認、プロファイル切替、CloudFormation操作、ログ確認の基本手順を整理します。

補足 AWS CLIの作業前には、利用者、アカウントID、リージョンを確認します。複数アカウントを扱う場合は、プロファイル名を明示して操作対象を取り違えないようにします。

インストール

作業端末のOSに合わせてAWS CLIをインストールします。

powershell
winget install --id Amazon.AWSCLI -e --source winget
aws --version
zsh
brew install awscli
aws --version
bash
curl "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" -o "awscliv2.zip"
unzip awscliv2.zip
sudo ./aws/install
aws --version

認証情報の確認

powershell
aws sts get-caller-identity

プロファイルの使い分け

powershell
aws configure --profile dev
aws sts get-caller-identity --profile dev

リージョン確認

作業前に対象リージョンを明示します。東京リージョンで作業する場合は ap-northeast-1 を指定します。

powershell
aws configure set region ap-northeast-1 --profile dev

CloudWatch Logs確認

powershell
aws logs describe-log-groups --profile dev
aws logs tail /aws/lambda/example-function --follow --profile dev

IAMユーザー・グループ操作

原則として人のログイン用途はIAM Identity Centerを優先します。IAMユーザーを作成する場合は、用途、権限、利用期限、アクセスキーの有無を事前に確認します。

powershell
aws iam create-user --user-name <USER_NAME> --profile dev
aws iam add-user-to-group --user-name <USER_NAME> --group-name <GROUP_NAME> --profile dev
aws iam list-groups-for-user --user-name <USER_NAME> --profile dev
確認観点 作業前後でユーザー数、所属グループ、付与ポリシーを比較します。アクセスキーを作成する場合は、保管方法、ローテーション、削除予定日を作業記録に残します。

Security Group ルール追加・削除

Security Group変更は影響範囲が広いため、対象SG、方向、プロトコル、ポート、送信元/宛先、説明欄を事前に確定します。

powershell
aws ec2 describe-security-groups --group-ids <SECURITY_GROUP_ID> --profile dev

aws ec2 authorize-security-group-ingress `
  --group-id <SECURITY_GROUP_ID> `
  --ip-permissions IpProtocol=tcp,FromPort=<PORT>,ToPort=<PORT>,IpRanges='[{CidrIp=<CIDR>,Description="<DESCRIPTION>"}]' `
  --profile dev

aws ec2 revoke-security-group-ingress `
  --group-id <SECURITY_GROUP_ID> `
  --protocol tcp `
  --port <PORT> `
  --cidr <CIDR> `
  --profile dev

SNS Topic・Subscription作成

通知先を作成する場合は、Topic、Subscription、確認メールの承認、通知テストまでを一連の手順として扱います。

powershell
aws sns create-topic --name <TOPIC_NAME> --profile dev
aws sns subscribe --topic-arn <TOPIC_ARN> --protocol email --notification-endpoint <MAIL_ADDRESS> --profile dev
aws sns publish --topic-arn <TOPIC_ARN> --message "notification test" --profile dev

Route 53 レコード確認

DNS変更はTTL、現在値、変更後の値、切り戻し値を事前に控えます。TXT/SPFレコードでは引用符、空白、includeの重複に注意します。

powershell
aws route53 list-resource-record-sets --hosted-zone-id <HOSTED_ZONE_ID> --profile dev
Resolve-DnsName -Name <RECORD_NAME> -Type TXT
