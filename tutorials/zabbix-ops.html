Zabbix 運用手順

Zabbix 6.0を想定し、Linux / RHEL9とWindows ServerのZabbix Agent 2導入、ホストグループ作成、メンテナンス設定、ホスト登録、最新データ確認を整理します。

前提 実環境のZabbix URL、顧客名、案件名、固定IPアドレスは扱わず、再利用できる手順として一般化しています。<HOSTNAME>、<ZABBIX_SERVER_OR_PROXY>、<PROJECT_NAME> などは環境に合わせて置き換えます。

1. 対象作業

作業確認観点
Agent導入Zabbix Agent 2のバージョン、Server / ServerActive、ListenPort、PSK設定、サービス状態を確認します。
ホストグループ作成運用単位、案件単位、環境単位で識別できる命名にします。
メンテナンス設定構築中または作業中の不要通知を抑止するため、対象ホストグループと期間を設定します。
ホスト登録Host名、Template、Interface、PSK、Port、所属グループを確認します。
最新データ確認Agent Ping、CPU、Diskなどの取得時刻と値が更新されていることを確認します。

2. Zabbix Agent 2インストール（RHEL9）

Zabbix 6.0のRHEL9向け公式リポジトリを追加します。dnf install だけでは通常Zabbix Agent 2を導入できないため、先に zabbix-release パッケージを導入します。

root shell
rpm -Uvh https://repo.zabbix.com/zabbix/6.0/rhel/9/x86_64/zabbix-release-6.0-5.el9.noarch.rpm

dnfのキャッシュを削除し、追加したリポジトリ情報を読み直します。

root shell
dnf clean all

Zabbix Agent 2をインストールします。

root shell
dnf install -y zabbix-agent2

3. Zabbix Agent 2設定（RHEL9）

Agent設定ファイルを編集します。環境ごとにZabbix ServerまたはProxyのIPアドレス、監視対象ホスト名を置き換えます。

値説明
<ZABBIX_SERVER_IP>Zabbix ServerまたはProxyのIPアドレス。
<ZABBIX_SERVER_FQDN>Zabbix ServerまたはProxyのFQDN。IPで指定する場合は不要です。
<HOSTNAME>Zabbixに登録するHost名。管理画面側のHost名と一致させます。

設定ファイルを編集します。

root shell
vi /etc/zabbix/zabbix_agent2.conf

主な設定例です。パッシブチェックでは Server、アクティブチェックでは ServerActive を確認します。

conf
Server=<ZABBIX_SERVER_IP>
ServerActive=<ZABBIX_SERVER_IP>
Hostname=<HOSTNAME>

設定値を確認します。

root shell
grep -E '^(Server|ServerActive|Hostname)=' /etc/zabbix/zabbix_agent2.conf

4. Zabbix Agent 2確認（RHEL9）

Zabbix Agent 2のバージョンを確認します。

root shell
zabbix_agent2 -V

Zabbix Agent 2を自動起動に設定し、起動します。

root shell
systemctl enable --now zabbix-agent2

サービス状態を確認します。

root shell
systemctl status zabbix-agent2

パッシブチェックを利用する場合は、Agentが10050/tcpで待ち受けていることを確認します。

root shell
ss -lntp | grep 10050
通信要件 パッシブチェックではZabbix ServerからAgentの10050/tcpへ到達できる必要があります。アクティブチェックではAgentからZabbix Serverへ接続します。Security Group、OS Firewall、Network ACL、経路をあわせて確認します。

5. Windows向けZabbix Agent 2インストール

Windows ServerへZabbix Agent 2を導入する場合は、Zabbix公式ダウンロードページからWindows向けMSIを取得し、ウィザードでServer、Host名、PSKを設定します。

値説明
<ZABBIX_SERVER_OR_PROXY>Zabbix ServerまたはProxyのIPアドレス、またはFQDN。複数ある場合はカンマ区切りで指定します。
<HOSTNAME>Zabbixに登録するHost名。Windowsのホスト名と運用上の管理名を合わせます。
<ZABBIX_PSK_IDENTITY>PSK identity。例: <HOSTNAME>
<ZABBIX_PSK_VALUE>PSK値。安全な場所で生成し、認証情報管理ルールに従って保管します。
  1. Zabbix公式のAgentダウンロードページを開きます。
  2. OS Distributionに Windows、OS Versionに対象Windowsバージョン、Hardwareに amd64 を選択します。
  3. Zabbix Versionに 6.0 LTS、Encryptionに OpenSSL、Packagingに MSI を選択します。
  4. 表示された6.0系の最新MSIをダウンロードし、対象サーバーで実行します。
  5. Host nameに <HOSTNAME> を入力します。
  6. Zabbix server IP / DNS、Server or Proxy for active checksに <ZABBIX_SERVER_OR_PROXY> を入力します。
  7. Agent listen portは 10050 を指定します。
  8. PSKを利用する場合はEnable PSKを有効化し、PSK identityとPSK valueを入力します。
  9. インストール後、services.msc を開き、Zabbix Agent 2 のスタートアップ種類が自動、サービス状態が実行中であることを確認します。

PSK値を作成する例です。作業端末にOpenSSLがある場合に実行します。

shell
openssl rand -hex 64

6. ホストグループ作成

Zabbix管理画面で、監視対象をまとめるホストグループを作成します。

  1. Zabbix管理画面にサインインします。
  2. 「設定」から「ホストグループ」を開きます。
  3. 「ホストグループを作成」を選択します。
  4. グループ名に、環境や案件を識別できる名称を入力します。
  5. 作成後、ホストグループ一覧に表示されることを確認します。
命名例用途
<CUSTOMER_CODE>.<PROJECT_CODE>_<CUSTOMER_NAME>_<PROJECT_NAME>顧客、案件、環境をまとめて識別する場合。
<PROJECT_NAME>-<ENV>小規模な検証環境や学習環境で簡潔に識別する場合。

7. メンテナンス設定

構築中や運用開始前に不要な通知が出ないよう、対象ホストグループにメンテナンス期間を設定します。

  1. 「設定」から「メンテナンス」を開きます。
  2. 「メンテナンス期間の作成」を選択します。
  3. 名前に作業内容、日付、作業者がわかる名称を入力します。
  4. 開始日時と終了日時を設定します。
  5. 対象に、作成したホストグループを指定します。
  6. 保存後、メンテナンス一覧に表示されることを確認します。
命名例用途
<PROJECT_NAME>_運用開始前のため静観_yyyymmdd(<OPERATOR>)運用開始前の構築期間を静観する場合。
<PROJECT_NAME>_計画作業_yyyymmdd(<OPERATOR>)計画作業中の通知抑止として使う場合。

8. ホスト登録

Agent導入後、Zabbix管理画面で監視対象ホストを登録します。

  1. 「設定」から「ホスト」を開きます。
  2. 「ホストの作成」を選択します。
  3. Host nameに <HOSTNAME> を入力します。
  4. Templatesに対象OSや監視方式に合うテンプレートを追加します。例: 基本監視 Linux/Windows (Activeエージェント2)
  5. Groupsに作成済みのホストグループを追加します。
  6. InterfacesでAgentを追加し、IP addressに対象サーバーのプライベートIP、Portに 10050 を指定します。
  7. PSKを利用する場合は、暗号化設定でPSK identityとPSK valueを設定します。
  8. 保存後、ホスト一覧でStatusが有効になっていることを確認します。

9. 最新データ確認

ホスト登録後、最新データが取得されるまで1から5分程度待ってから確認します。

  1. 「監視データ」から「最新データ」を開きます。
  2. Hostに <HOSTNAME> を指定してフィルタします。
  3. Agent Ping、CPU使用率、Disk使用量などのアイテムが表示されることを確認します。
  4. Latest checkの時刻が更新され、値が取得されていることを確認します。

10. 通知先変更

通知先変更では、対象ユーザー、メディア種別、送信先、通知時間帯、有効/無効を作業前後で比較します。

  1. Zabbix管理画面にサインインします。
  2. 対象ユーザーまたはユーザーグループを開きます。
  3. メディア設定を確認し、作業前エビデンスを取得します。
  4. 通知先を追加または変更します。
  5. 更新後の設定と通知テスト結果を確認します。

11. エビデンス取得

タイミング取得対象
作業前対象Host、Template、Item、Trigger、Action、通知先、メンテナンス設定。
作業後変更後の設定、最新データ、通知テスト、イベント履歴。
切り戻し時変更前設定への復旧、通知停止または復旧確認。
