AWS / Infrastructure / IaC / Operations

Knowledge

CloudFormation、AWS CLI、Terraform、AWX / Ansible など、クラウドエンジニアの実務で使う手順を横断的に整理するためのドキュメントサイトです。

各GitHubリポジトリには構成図、CloudFormationテンプレート、パラメータ例を置き、このサイトにはリポジトリをまたいで使う運用手順や確認観点をまとめます。

運用方針 リポジトリは実装本体、Knowledge は横断手順集として分けます。READMEやdocsと重複させすぎず、変更作業、障害時の確認、ツール操作の手順を中心に整理します。

Guides

CloudFormation 運用手順集

変更セット、ロールバック、ドリフト検出、削除前確認など、運用時の事故を減らすための確認観点を整理します。

AWS CLI 実務手順集

プロファイル切替、認証確認、CloudFormation操作、ログ確認などの基本操作をまとめます。

Terraform 入門から運用まで

init、plan、apply、destroy、tfstate、backend、variables などを実務目線で整理します。

AWX / Ansible 手順

Inventory、Credential、Project、Job Template、Playbook実行ログの確認観点を整理します。

記法メモ

GitHub README/docsで使うMarkdown記法と、Pages側で使うHTML記法を整理します。

Repositoryとの使い分け

場所 役割
Profile README 自己紹介、代表リポジトリ、スキル概要を掲載します。
