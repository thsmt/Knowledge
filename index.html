GitHub Pages

GitHub Pages は、GitHub リポジトリ内の静的ファイルをそのまま公開できるホスティング機能です。 HTML、CSS、画像、Markdown を使って、ポートフォリオ、技術メモ、ハンズオン資料を軽量に公開できます。

このページは、GitHub Pages の公開確認と、今後のドキュメントサイト化を想定したテストページです。 左側にチュートリアル一覧、中央に本文、右側に目次を配置し、AWS や IaC の学習資料を整理しやすい構成にしています。

補足 現在のGitHubログイン名は hsmtth として公開されています。GitHub IDを変更した場合、Pages URLも新しいIDに追従します。

インストール

GitHub Pages 自体に追加インストールは不要です。ローカル確認を行う場合のみ、任意で簡易HTTPサーバーを起動します。

Windows

PowerShell でローカル確認用の簡易サーバーを起動します。

powershell 
cd .\github-pages-test
python -m http.server 8000

ブラウザで http://localhost:8000 を開きます。

macOS

zsh でローカル確認用の簡易サーバーを起動します。

zsh 
cd github-pages-test
python3 -m http.server 8000

Linux

任意のターミナルでローカル確認用の簡易サーバーを起動します。

bash 
cd github-pages-test
python3 -m http.server 8000

リポジトリ構成

本番のポートフォリオサイトでは、学習用 lab と業務検証に近い poc を分けて掲載すると、採用担当者が意図を読み取りやすくなります。

lab repositories

AWS学習、書籍ベースの検証、CloudFormationテンプレートを整理します。

poc repositories

業務に近い要件、設計判断、運用観点、手動手順を整理します。

デプロイ

GitHub Pages は main ブランチのルートを公開元に設定しています。 `index.html` と `assets/` を更新して push すると、自動的に公開ページへ反映されます。

git 
git add -A
git commit -m "Update GitHub Pages test site"
git push
Repository: hsmtth/github-pages-test
